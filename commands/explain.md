Explain how the specified code works.

Target: $ARGUMENTS

## Approach

1. **Read the code** thoroughly — the full file and its imports/dependencies
2. **Explain at multiple levels**:
   - **High level**: What does this code accomplish? What problem does it solve?
   - **Architecture**: How is it structured? What are the key components and how do they interact?
   - **Flow**: Walk through the execution path step by step
   - **Key details**: Important algorithms, data structures, or patterns used
3. **Highlight non-obvious things**: Gotchas, clever tricks, implicit dependencies, potential pitfalls
4. **Use diagrams** (ASCII) if the architecture is complex
5. **Keep it concise** — explain what matters, skip the obvious
